# Base Oficial de Conhecimento Operacional — Orçamentos

## Objetivo

Esta base preserva o conhecimento adquirido sobre como a FOS compreende problemas, levanta informações, toma decisões e elabora orçamentos.

Seu conteúdo representa conhecimento operacional da FOS. Ele deve permanecer útil independentemente da tecnologia, da arquitetura ou da implementação adotada pelo APP FOS.

## Documentação operacional e arquitetura

A documentação operacional registra **como a FOS trabalha**: práticas observadas, informações necessárias, decisões, exceções e aprendizados do processo de orçamento.

A documentação de arquitetura registra **como o software é estruturado** para apoiar esse trabalho.

Um finding operacional não define automaticamente entidade, tela, campo, fórmula, fluxo de interface ou mecanismo de persistência. Qualquer tradução para arquitetura ou implementação exige etapa própria, baseada nas evidências acumuladas.

## Regras permanentes

1. Cada finding deve possuir arquivo próprio e identificação única.
2. Findings são cumulativos e formam histórico permanente.
3. Conhecimento registrado nunca deve ser apagado.
4. Correções, ampliações ou contrapontos devem ser registrados por novos findings, preservando os anteriores.
5. Todo finding e toda alteração desta base devem ser versionados no repositório.
6. O índice oficial deve ser atualizado quando um novo finding for acrescentado.
7. Fatos observados devem permanecer separados de interpretações e decisões futuras.
