# Checklist de Homologação das Análises de Orçamento

## Antes da alteração

- [ ] Ler `docs/PROJECT_STATE.md`.
- [ ] Ler `docs/DEVELOPMENT_PHILOSOPHY.md`.
- [ ] Ler `docs/architecture/README.md`.
- [ ] Ler toda documentação relacionada ao orçamento analisado.
- [ ] Verificar consistência e possíveis duplicidades.
- [ ] Identificar impactos documentais.
- [ ] Confirmar a necessidade da alteração.

## Cobertura da análise

- [ ] O nome completo do Excel, incluindo extensão, identifica a análise.
- [ ] Todas as abas foram inventariadas.
- [ ] Cada aba possui objetivo, papel, entradas, saídas, dependências, entidades, regras, lógica das fórmulas, variáveis, observações e dúvidas.
- [ ] O fluxo completo da planilha foi descrito.
- [ ] Fórmulas foram explicadas por finalidade.
- [ ] Dependências entre abas foram registradas.
- [ ] Erros, exceções e inconsistências foram preservados.
- [ ] Fatos, informações do especialista, interpretações e hipóteses foram separados.
- [ ] Descobertas relevantes receberam nível de confiança A, B, C ou D.
- [ ] Nenhuma arquitetura ou implementação foi proposta nesta etapa.

## Homologação

- [ ] Dúvidas relevantes foram apresentadas ao especialista.
- [ ] Correções recebidas foram registradas com rastreabilidade.
- [ ] Hipóteses permaneceram explicitamente identificadas.
- [ ] O status da análise foi declarado.
- [ ] A análise não foi consolidada automaticamente.

## Após a alteração

- [ ] A escrita foi confirmada pela ferramenta.
- [ ] O documento foi reaberto.
- [ ] O conteúdo gravado foi conferido.
- [ ] A consistência com os documentos relacionados foi confirmada.
- [ ] O commit correspondente foi registrado.
